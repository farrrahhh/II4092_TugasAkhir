Kedua model ini menjadi dasar implementasi sistem secara keseluruhan.

\subsection{Use Case}

Pada sistem aplikasi pembelajaran Algoritma dan Struktur Data berbasis \textit{guided learning} ini, terdapat satu aktor yaitu \textit{user} yang merupakan mahasiswa sebagai pengguna sistem. \textit{User} berinteraksi dengan sistem untuk mempelajari materi ASD secara terstruktur, mulai dari memilih topik, memahami konsep, melihat contoh, mengerjakan latihan, hingga memantau progres belajar.

Sistem dirancang sebagai \textit{teman belajar digital} yang mendampingi mahasiswa dalam proses pembelajaran. Melalui integrasi LLM, sistem mampu membangkitkan soal latihan yang adaptif, mengevaluasi jawaban pengguna secara otomatis, serta memberikan umpan balik yang dipersonalisasi terhadap latihan yang dikerjakan oleh \textit{user}.

Tabel \ref{tab:use-case} menunjukkan bahwa terdapat satu aktor yaitu \textit{user} yang dapat menjalankan berbagai aktivitas untuk memenuhi kebutuhan fungsional sistem. Proses pembelajaran dimulai dari \textit{user} memilih topik ASD yang ingin dipelajari (UC-01).

Selanjutnya, \textit{user} akan memasuki alur pembelajaran terstruktur dengan melihat materi (UC-02) yang berisi penjelasan konsep yang dihasilkan sistem. Untuk memperdalam pemahaman, \textit{user} dapat melihat contoh implementasi (UC-03) sebelum mengerjakan latihan yang diberikan (UC-04).

Setelah menyelesaikan latihan, \textit{user} dapat melihat pembahasan (UC-05) sebagai bentuk umpan balik dari sistem. Relasi \textit{extend} antara UC-04 dan UC-05 menunjukkan bahwa pembahasan hanya muncul setelah \textit{user} mengerjakan latihan. Selain itu, \textit{user} juga dapat melihat ringkasan materi (UC-06) untuk memperkuat pemahaman, serta memantau progres belajar melalui fitur \textit{progress} (UC-07).

Sebagai pelengkap alur utama, \textit{user} dapat memanfaatkan dua fitur tambahan yang tersedia pada setiap halaman topik: mengevaluasi soal mandiri (UC-08) dengan memasukkan soal dan jawaban dari sumber eksternal untuk dinilai oleh sistem, serta bertanya kepada asisten materi (UC-09) melalui widget percakapan kontekstual yang memberikan penjelasan sesuai topik yang sedang dipelajari.

Dengan demikian, seluruh \textit{use case} yang dirancang telah mencerminkan alur \textit{guided learning} yang mendukung pembelajaran ASD secara terstruktur dan interaktif, serta memenuhi kebutuhan fungsional sistem.

\subsection{Arsitektur Sistem}

Sistem Teman Belajar ASD dirancang menggunakan arsitektur berlapis yang terdiri atas empat komponen utama: \textit{Client}, \textit{Frontend}, \textit{Backend}, serta \textit{Database} dan OpenAI API. \textit{Client} mengakses sistem melalui \textit{browser} atau perangkat \textit{mobile} via HTTPS. \textit{Frontend} yang dibangun dengan Next.js berperan sebagai antarmuka pengguna sekaligus \textit{proxy} ke \textit{backend}, menangani halaman autentikasi, \textit{dashboard} topik, alur pembelajaran, dan \textit{progress tracker}. \textit{Backend} berbasis FastAPI mengelola seluruh logika bisnis meliputi autentikasi, progres belajar, generasi soal latihan, evaluasi jawaban, dan \textit{adaptive engine}. Data pengguna disimpan di PostgreSQL, sedangkan generasi konten pembelajaran dilakukan melalui OpenAI API (GPT-4o-mini). Seluruh layanan di-\textit{deploy} di GCP Cloud Run dengan \textit{pipeline} CI/CD melalui GitHub Actions.

\section{Perancangan \textit{Website}}

Bagian ini membahas perancangan teknis aplikasi web Teman Belajar ASD, meliputi arsitektur sistem dan tumpukan teknologi yang digunakan, spesifikasi \textit{endpoint} API, rancangan antarmuka pengguna, serta konfigurasi \textit{hosting} dan \textit{deployment}.

\subsection{Perancangan Arsitektur Sistem}

Aplikasi Teman Belajar ASD dibangun menggunakan arsitektur \textit{client-server} yang terdiri dari tiga lapisan utama: \textit{frontend}, \textit{backend}, dan \textit{database}. Komponen \textit{frontend} dibangun dengan Next.js dan berkomunikasi dengan \textit{backend} melalui RESTful API berbasis FastAPI. \textit{Backend} menangani logika bisnis, autentikasi JWT, serta integrasi dengan OpenAI API untuk membangkitkan soal latihan, mengevaluasi jawaban pengguna, dan melayani chatbot asisten materi. Data pengguna dan progres belajar disimpan pada PostgreSQL. Tabel \ref{tab:tech-stack} menyajikan komponen teknologi yang digunakan pada setiap lapisan sistem.

\subsection{Spesifikasi \textit{Hosting}}

Aplikasi di-\textit{deploy} menggunakan Google Cloud Run sebagai \textit{platform hosting} berbasis kontainer yang bersifat \textit{serverless}. \textit{Frontend} dan \textit{backend} masing-masing dikemas dalam \textit{Docker image} dan dijalankan sebagai layanan Cloud Run yang terpisah. \textit{Database} menggunakan Google Cloud SQL for PostgreSQL yang terhubung ke \textit{backend} melalui Cloud SQL Connector. Seluruh proses \textit{build} dan \textit{deployment} dijalankan secara otomatis melalui GitHub Actions setiap kali terdapat \textit{push} ke \textit{branch} utama.

\textit{Deployment} dilakukan secara otomatis melalui integrasi dengan repositori Git. Setiap \textit{push} ke \textit{branch} utama akan memicu \textit{pipeline} GitHub Actions yang mendeteksi perubahan pada direktori \texttt{frontend/} atau \texttt{backend/} secara terpisah. \textit{Image} Docker di-\textit{build} dan dikirim ke Google Artifact Registry, kemudian di-\textit{deploy} ke layanan Cloud Run yang sesuai. Autentikasi ke Google Cloud menggunakan Workload Identity Federation sehingga tidak memerlukan \textit{service account key} yang disimpan secara statis.
